How to Use
Basic workflow
  • Pick a source, then press Play source. Load audio adds a file and starts it.
  • Drag to orbit, mouse wheel to zoom, Shift-drag to pan.
  • Use Top view to see pitch classes around the helix.
Analysis
  • FFT peaks: good general-purpose mode for recorded music and rich sounds.
  • Constant Q: log-frequency bands; useful for musical pitch structure.
  • Note bins: exact equal-tempered note detectors; best for clean tests or stable pitched music.
Normalization
  • Frame: always makes the current loudest component bright. Good for debugging, bad for decay.
  • Adaptive: recent loudness reference. Good for mic/live input.
  • Track: fixed reference for the whole file. Best for preserving song dynamics.
Show
  • Spectrogram shows the analysis history at the right.
  • Notes labels pitch classes under the helix and on the spectrogram.
  • Octaves shows reference rings at octave heights.
Tuning
  • Sensitivity raises or lowers how easily points appear.
  • Fade controls how long old points remain visible.
  • Contrast controls harmonic rolloff: lower values make weak partials disappear faster.
Sync

Analysis uses a time window, so visuals can feel late. Audio delay delays what you hear so sound and light line up. Try 80-150 ms for Note bins and 180-350 ms for FFT or Constant Q.

Note tests

Test A note shows a single fundamental plus expected harmonics. Test A chord and the note grid use clean sine fundamentals, which is useful for checking that pitch placement is correct.

Advice

There is no single perfect setting. Try FFT peaks or Constant Q for real music, Note bins for clean note-centered experiments, and Track normalization when playing loaded songs.
